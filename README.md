## Мета роботи

Вивчення стандарту HTML5 та CSS, способів виконання базових операцій, особливостей створення вебсторінок з використанням каскадних таблиць стилів.

## Структура проекту

```
lab_01/
├── index.html       ← Головна сторінка (Про довідник, класифікація, рослина місяця)
├── plants.html      ← Каталог рослин (квіткові, дерева, трави, таблиця, iframe)
├── contact.html     ← Контакти, медіа (аудіо + відео), форма зворотного зв'язку
├── style.css        ← Єдиний файл стилів для всіх сторінок
└── media/           ← Папка для медіафайлів (mp3, mp4, webm, poster.jpg)
    ├── nature-sounds.mp3
    ├── nature-sounds.ogg
    ├── botanical-garden.mp4
    ├── botanical-garden.webm
    ├── botanical-garden.ogv
    └── garden-poster.jpg
```

> **Примітка щодо медіафайлів:** Файли в папці `media/` потрібно додати самостійно.  
> Без них теги `<audio>` та `<video>` відображаться, але не відтворюватимуть контент.

## Контрольні запитання (непарний варіант — непарні питання)

**1. Що таке браузер?**  
Браузер — програмний застосунок, призначений для доступу до інформації в Інтернеті. Він інтерпретує HTML-код сторінки, знаходить теги розмітки та відображає їх у вигляді стильового оформлення змісту.

**3. Опишіть загальний синтаксис мови HTML.**  
Синтаксис HTML базується на тегах: `<тег атрибут="значення">` (одиночний) або `<тег атрибут="значення">...вміст...</тег>` (парний). Теги бувають одиночні (self-closing) та парні (контейнери). Атрибути — обов'язкові та необов'язкові пари ключ/значення всередині відкриваючого тегу.

**5. Для чого потрібен тег `<head>`?**  
Тег `<head>` призначений для зберігання службових елементів документа: мета-тегів, заголовку `<title>`, підключення стилів `<link>`, скриптів `<script>`. Вміст `<head>` не відображається на сторінці.

**7. Які властивості елементу `<div>`?**  
`<div>` — універсальний блоковий контейнер. Завжди відокремлюється від інших елементів порожнім рядком, не несе семантичного навантаження (є роздільником), дозволяє застосовувати CSS-атрибути, пов'язані з межею блоків.

**9. Які існують способи використання стилів у документі?**  
1. Зовнішній файл стилів (підключається через `<link rel="stylesheet">`).  
2. Внутрішні стилі в `<style>` всередині `<head>`.  
3. Вбудовані стилі через атрибут `style=""` безпосередньо в тезі.

**11. Що таке ідентифікатор у CSS?**  
Ідентифікатор (ID-селектор) визначає унікальне ім'я елемента, яке використовується для зміни його стилю. Синтаксис: `#ім'я { властивість: значення; }`. В HTML задається атрибутом `id=""`. Кожен ID унікальний у межах документа.

**13. За допомогою яких тегів можна розділити сторінку на частини?**  
`<header>` (шапка), `<nav>` (навігація), `<main>` (основний вміст), `<article>` (стаття), `<section>` (розділ), `<aside>` (бічна панель), `<footer>` (підвал), `<div>` (універсальний блок).

**15. Назвіть основні атрибути валідації форми.**  
`required` — обов'язкове поле; `maxlength` — максимальна довжина тексту; `min` / `max` — діапазон числових/дата-значень; `step` — крок зміни значення; `pattern` — регулярний вираз для перевірки формату.

**17. Для чого потрібен тег `<video>`? Назвіть атрибути цього тегу.**  
`<video>` додає, відтворює і управляє відеороликом на вебсторінці. Основні атрибути: `src` (шлях до файлу), `controls` (елементи управління), `autoplay`, `loop`, `muted`, `poster` (зображення-заставка), `width`, `height`. Вкладений тег `<source>` дозволяє вказати кілька форматів.

**19. Які є головні етапи створення сайту?**  
1. Визначення мети та цільової аудиторії.  
2. Планування структури та карти сайту.  
3. Розробка дизайну (wireframe, макет).  
4. Верстка HTML/CSS.  
5. Програмування функціоналу (JS, backend).  
6. Наповнення контентом.  
7. Тестування у різних браузерах.  
8. Публікація (хостинг, домен).
